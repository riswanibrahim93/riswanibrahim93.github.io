Web Appointment Booking Testing

Project Overview

This project focuses on testing a web-based appointment booking application that includes core features such as login, appointment creation, history management, and logout.

The objective of this project is to ensure the reliability of core functionalities and improve testing efficiency through automation.

I was responsible for designing test strategies, developing test cases, and implementing an automation framework to support scalable and maintainable test execution.

Problem - Solution - Impact
  • Lack of Flexibility & Reusability

    Problem: Automation tests were not flexible and required code changes for different scenarios, while test scripts were repetitive and difficult to maintain.

    Solution: Designed a modular hybrid automation framework (data-driven & keyword-driven) with separation of scripts, test data, and locators.

    Impact: Improved reusability, reduced maintenance effort, and enabled scalable automation.

  • Automation Instability (Timing & Locator Issues)

    Problem: Automation tests were unstable due to timing issues and frequently changing locators, causing flaky test results.

    Solution: Enhanced synchronization using explicit waits and improved locator strategies with centralized management.

    Impact: Improved test stability, reduced flaky failures, and increased reliability.

Test Strategy

The testing approach combines manual and automation testing, focusing on core functionalities, input validation, and edge case scenarios to ensure application reliability.

  • Core functionality testing (Login & Booking)
  • Validation and negative testing
  • Edge case handling

Automation is prioritized based on feature usage, business impact, system stability, and implementation effort.

Test Scenarios

Login
  • Valid and invalid login attempts
  • Empty input validation
  • Invalid character handling
Appointment
  • Booking with valid data
  • Validation for required fields
  • Edge case (past date)
History
  • View booking history
  • Empty state handling
Logout
  • Successful logout flow
